# Gmail OAuth 2.0

## Two Files to Know

- `~/.config/ai-employee/client_secret.json` — downloaded from Google Cloud Console (do not commit)
- `~/.config/ai-employee/gmail_credentials.json` — generated by the auth flow, contains `access_token` + `refresh_token` (auto-refreshed by the library)

## Setup Flow (one-time per account)

### 1. Google Cloud Console
1. Create project at [console.cloud.google.com](https://console.cloud.google.com)
2. Enable **Gmail API** (APIs & Services → Library)
3. Enable **Google Calendar API** if Calendar integration is needed (same place)
4. Create OAuth 2.0 credentials: APIs & Services → Credentials → Create Credentials → **OAuth client ID** → Desktop App
5. Download the JSON → save to `~/.config/ai-employee/client_secret.json`
6. OAuth consent screen → add Gmail address as a **Test User** (required while app is unverified)

### 2. Run the OOB Auth Flow (WSL2 / headless)

> **WSL2 note:** `run_local_server()` doesn't work in WSL2 because there's no browser. Use the OOB flow below — it prints a URL, you open it in Windows browser, paste the code back.

```bash
uv run python - << 'PYEOF'
from pathlib import Path
from google_auth_oauthlib.flow import InstalledAppFlow

SCOPES = [
    "https://www.googleapis.com/auth/gmail.readonly",
    "https://www.googleapis.com/auth/gmail.send",
    "https://www.googleapis.com/auth/gmail.modify",
    # Add calendar scopes here if needed (see google-calendar skill):
    # "https://www.googleapis.com/auth/calendar.readonly",
    # "https://www.googleapis.com/auth/calendar.events",
]

flow = InstalledAppFlow.from_client_secrets_file(
    str(Path.home() / ".config/ai-employee/client_secret.json"), SCOPES
)
flow.redirect_uri = "urn:ietf:wg:oauth:2.0:oob"
auth_url, _ = flow.authorization_url(prompt="consent")
print("Open in browser:", auth_url)
code = input("Paste authorization code: ").strip()
flow.fetch_token(code=code)
out = Path.home() / ".config/ai-employee/gmail_credentials.json"
out.write_text(flow.credentials.to_json())
print("Saved to", out)
PYEOF
```

Open the printed URL → sign in → grant permissions → paste the code shown by Google.

## Required Scopes

```python
# Gmail only
SCOPES = [
    "https://www.googleapis.com/auth/gmail.readonly",
    "https://www.googleapis.com/auth/gmail.send",
    "https://www.googleapis.com/auth/gmail.modify",
]

# Gmail + Calendar (single combined token — recommended)
SCOPES = [
    "https://www.googleapis.com/auth/gmail.readonly",
    "https://www.googleapis.com/auth/gmail.send",
    "https://www.googleapis.com/auth/gmail.modify",
    "https://www.googleapis.com/auth/calendar.readonly",
    "https://www.googleapis.com/auth/calendar.events",
]
```

**If scopes change:** delete `gmail_credentials.json` and re-run the OOB flow — the refresh token won't grant new scopes automatically.

## Token Refresh (automatic)

```python
# src/mcp_servers/gmail_service.py:26-40
creds = Credentials.from_authorized_user_file(str(creds_path))
if creds.expired and creds.refresh_token:
    from google.auth.transport.requests import Request
    creds.refresh(Request())
    creds_path.write_text(creds.to_json(), encoding="utf-8")  # persist refreshed token
service = build("gmail", "v1", credentials=creds)
```

Token refresh is silent and automatic as long as `refresh_token` exists in the credentials JSON. Refresh tokens don't expire unless revoked.

## Send Email

```python
from email.mime.text import MIMEText
import base64

msg = MIMEText(body)
msg["to"] = "recipient@example.com"
msg["subject"] = "Subject line"
raw = base64.urlsafe_b64encode(msg.as_bytes()).decode()
result = service.users().messages().send(userId="me", body={"raw": raw}).execute()
message_id = result["id"]
```

For replies, add headers:
```python
msg["In-Reply-To"] = original_message_id
msg["References"] = original_message_id
```

```python
    # full["snippet"] — first ~100 chars of body
```

Gmail query syntax: `from:`, `to:`, `subject:`, `is:unread`, `after:2024/01/01`, `has:attachment`

## Attachment

```python
from email.mime.multipart import MIMEMultipart
from email.mime.base import MIMEBase
import mimetypes

msg = MIMEMultipart()
msg.attach(MIMEText(body, "plain"))
content_type, _ = mimetypes.guess_type(file_path)
main_type, sub_type = (content_type or "application/octet-stream").split("/", 1)
part = MIMEBase(main_type, sub_type)
part.set_payload(Path(file_path).read_bytes())
part.add_header("Content-Disposition", "attachment", filename=Path(file_path).name)
msg.attach(part)
```

## Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `invalid_grant` | Refresh token revoked or expired | Re-run the OOB auth flow above — paste new code |
| `insufficientPermissions` | Missing scope | Delete `gmail_credentials.json`, re-run OOB flow with correct SCOPES |
| `FileNotFoundError` on creds path | Wrong `GMAIL_CREDENTIALS_PATH` | Run OOB flow first, check path in .env |
| `Access blocked: app not verified` | Test user not added | Add email to OAuth consent screen Test Users |
| `quota exceeded` | Too many API calls | Gmail API has daily quota; cache results, reduce polling |
| Calendar API 403 `accessNotConfigured` | Calendar API not enabled | Go to GCP Console → APIs & Services → Library → enable "Google Calendar API" for the correct project. Note: project number shown in error URL |
