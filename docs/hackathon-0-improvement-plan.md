### Madam Wania Kazmi — Agent Maturity Model (Day 3)

Presented the conceptual framework for the hackathon series:

- **Phase 1 (General Agent)**: Claude Code doing everything — good for prototyping, non-deterministic, expensive
- **Phase 2 (Custom Agent)**: Specialized, deterministic, deployed — uses OpenAgent SDK or custom framework
- **Transition path**: Use general agent to incubate + design the custom agent step-by-step

**Our gap**: We are 100% Phase 1. No custom agent harness, no OpenAgent SDK, no specialization.

## Improvement Plan (Prioritized)

### Priority 1 — High Impact, Low-Medium Effort (DO FIRST)

**1. Tmux session persistence**
- Add `scripts/start.sh` that launches the agent inside a named Tmux session
- Any Claude sub-session spawned by orchestrator should also use Tmux panes
- Files: create `scripts/start.sh`, update `src/orchestrator/orchestrator.py`

**2. Gmail label tagging**
- After processing each email, apply Gmail labels: `AI/processed`, `AI/not-for-ai`, `AI/replied`
- Makes inbox self-organizing — clearly shows what agent touched
- File: `src/mcp_servers/gmail_service.py`

**3. Claude Code hooks for notifications**
- Create `.claude/hooks/` with scripts for `on_task_complete`, `on_task_failed`, `on_task_pending`
- Fire terminal/desktop notifications whenever agent completes or gets blocked
- Very visible in demos — makes the agent feel alive

**4. Google Push Notifications for Gmail**
- Replace polling in `src/watchers/gmail_watcher.py` with Google PubSub webhook
- Google fires event immediately → zero latency, no rate limit issues
- File: `src/watchers/gmail_watcher.py`

### Priority 2 — Medium Impact, Medium Effort

**5. Google Calendar integration**
- Add `src/mcp_servers/google_calendar_client.py` + `calendar_mcp.py`
- Agent reads upcoming events, creates events, schedules its own recurring work
- Wire into orchestrator so agent can book time blocks

**6. Dynamic self-scheduling**
- Expand `scheduler.py` so agent can schedule its own future tasks at runtime
- E.g., agent decides "send follow-up in 2 hours" → writes entry → fires later

**7. CEO Weekly Briefing**
- Auto-generate weekly summary: tasks completed, emails handled, social posts, metrics
- Send via email + WhatsApp every Sunday
- Judges clearly look for this (Junaid + Ruhubhai both had it)

**8. Concurrency audit**
- Verify `orchestrator.py` uses thread pooling for parallel task execution
- Multiple inbox tasks must start simultaneously, not queue sequentially
- Add explicit concurrency config (max_workers, timeout_per_task)

**9. Process self-healing audit**
- Verify `health_monitor.py` detects + restarts stuck RALF loops
- Watcher should auto-clean orphaned Claude sub-processes

### Priority 3 — Polish + Phase 2 Transition

**10. Twitter wiring audit**
- Verify `twitter_client.py` is dispatched from the orchestrator harness
- Agent should post, reply, schedule tweets autonomously without manual trigger

**11. Multi-LLM config**
- Add `model_config.yaml` — swap underlying LLM via config
- Not vendor-locked: Claude in front, anything else in back

**12. Lead qualification chatbot (Odoo)**
- Wire `odoo_mcp.py` into a chatbot that scores leads (warm/hot/cold)
- On new lead → auto send welcome email + WhatsApp

**13. Custom agent skeleton (Phase 2)**
- Start OpenAgent SDK integration for one specific role (e.g., Social Media Manager)
- Move from general agent to deterministic, deployable custom agent
- This is what Hackathon 5 will require

Based on what we've built, what the judges saw, and the trajectory of Hackathon 5 (custom agents), here are   
  the next-level improvements:                                                                                
                                                                                                                
  ---                                                                                                           
  Tier 1 — High Impact, Demo-Ready                                                                              
                                                                                                                
  1. Inbound WhatsApp → Task Creation

  Right now WhatsApp is outbound only. If someone sends a WhatsApp message to the AI Employee number, it should:
  - Parse the message as a task or question
  - Create a Needs_Action file
  - Reply with acknowledgement
  - Complete and reply back

  This makes the agent feel truly interactive — not just a scheduler.

  2. Meeting Scheduler (Email → Calendar)

  When someone emails "let's meet this week" or "can we schedule a call?":
  - Agent reads the email
  - Checks Google Calendar for free slots
  - Drafts a reply with 2–3 time options
  - On acceptance → creates calendar event + sends confirmation

  Sir Junaid had Calendar but didn't show this flow. This would be unique.

  3. Live Web Dashboard (Real-Time)

  Currently web_dashboard.py exists but isn't clearly a live-updating UI. Replace with a proper FastAPI +
  WebSocket dashboard showing:
  - Active tasks (in-progress, pending approval, done)
  - Today's schedule from Google Calendar
  - Email stats (processed, replied, tagged)
  - Social media posts queued/sent
  - Agent heartbeat (is it alive?)

  This is the most visible demo element — judges see it in 10 seconds.

  4. PDF/Attachment Processing

  When an email arrives with a PDF attachment:
  - Agent downloads and parses it
  - Extracts key information and action items
  - Creates a structured Needs_Action entry with summary

  This is a very common real-world need and no one in the transcripts showed it.

  ---
  Tier 2 — Intelligence Improvements

  5. Agent Memory & Personalization

  The agent currently has no long-term memory of contacts or preferences. Add a contacts_memory.json in the
  vault:
  - Remember how each contact prefers to be addressed
  - Remember what topics they care about
  - Remember past interactions and outcomes
  - Use this when composing emails and social posts

  6. Proactive Follow-Up Engine

  Scan Done/ folder and identify tasks that should have a follow-up:
  - "Sent proposal to X 3 days ago — no response" → draft follow-up
  - "Onboarded client Y last week" → send check-in message
  - Runs daily as a scheduled task

  This demonstrates proactive behavior — the agent acts without being asked.

  7. Content Calendar (Social Media)

  Instead of one-off posts, the Social Media Agent plans a full week:
  - Sunday: generate 5–7 posts for the week across platforms
  - Write them to a Content_Calendar.md in vault
  - Schedule them via scheduler.schedule_once_after()
  - Post automatically at optimal times

  8. Inbound Email Auto-Draft (Not Auto-Send)

  For every email that arrives:
  - Agent drafts a reply and saves it as a Gmail Draft
  - Moves email to AI/drafted label
  - User reviews and clicks Send in Gmail normally

  This is safer than auto-send and still saves significant time. The approval flow is Gmail's native UI instead
  of vault files.

  ---
  Tier 3 — Architecture / Phase 2

  9. Multi-Agent Coordination

  Deploy two specialized agents working together:
  - Sales Agent: handles leads, follows up, creates Odoo records
  - Content Agent: handles social media, drafts emails
  - They share the vault for coordination (claim-by-move rule already implemented)

  This directly demonstrates Madam Wania's Phase 2 custom agent model.

  10. Self-Performance Review

  Every Sunday alongside the CEO briefing:
  - Agent counts its own metrics: tasks completed, emails handled, posts published, errors
  - Compares against last week
  - Writes a Performance_Review.md
  - Suggests: "I handled 12 emails this week vs 8 last week — 50% improvement"

  This is the self-awareness element Sir Junaid hinted at.

  11. Voice Message Handling (WhatsApp)

  WhatsApp supports voice notes. When a voice message arrives:
  - Use Whisper API to transcribe
  - Treat transcription as a task/command
  - Reply via text

  Extremely impressive in a demo — speak to your AI Employee on WhatsApp.

  12. Budget Alert System

  Monitor Odoo/ledger daily:
  - Alert via WhatsApp if any category exceeds budget threshold
  - Weekly spend summary alongside CEO briefing
  - "Your AWS bill this month is 40% over budget"
